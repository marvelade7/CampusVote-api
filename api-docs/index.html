Introduction

Welcome to the CampusVote API documentation. This API powers the voting and election platform, providing robust authentication, election management, and candidate applications.

Base URL

All API requests should be prefixed with the following base URL. Replace 5000 with your configured PORT if running locally.

http://localhost:5000/api

Authentication

The CampusVote API uses JSON Web Tokens (JWT) for authentication. Upon successful login or registration, you will receive a token in the response.

For protected routes, you must include this token in the Authorization header of your HTTP request:

Authorization: Bearer <your_jwt_token>
POST

/users/signup

Registers a new student user onto the platform. Requires personal details and a unique matriculation number.

Request Body

Field Type Required Description
firstName string Yes User's first name
lastName string Yes User's last name
email string Yes Valid email address (Unique)
password string Yes Minimum 6 characters
confirmPassword string Yes Must match password
matricNumber string Yes Student matriculation number (Unique)
faculty string Yes e.g., "Engineering"
department string Yes e.g., "Computer Science"
level number Yes Enum: 100, 200, 300, 400, 500, 600
termsAccepted boolean Yes Must be true

POST

/users/login

Authenticates an existing user and returns an access token. Users can login using either their email or matriculation number.

Request Body

Field Type Required Description
identifier string Yes Email OR Matric Number
password string Yes Account password

  "identifier": "johndoe@example.com",
  "password": "password123"

Database Models

To help you understand the backend's data structure, here are the main entities (schemas) defined in the MongoDB database:

👤

User

Represents users of the system. Roles include student and admin. Stores core identity like matric number, department, level, and authentication details.

🗳️

Election

Represents an election event. Tracks status (upcoming, ongoing, completed), active dates, allowed voters list, and participating candidates.

🎓

Candidate

Represents a user running for a specific position in an election. Contains details like their manifesto, the position they are contesting for, and their vote tally.

📝

Contest Application

Used when a student applies to become a candidate. Tracks application status (pending, approved, rejected), their motivation statement, and the reviewer.
